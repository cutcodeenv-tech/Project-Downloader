# Архитектура проекта

## Назначение

MVP-фреймворк для автоматизации производства документальных видео серии "Основатели" о российских предпринимателях. Автоматизирует сбор, обработку и подготовку медиа-контента для монтажа в Adobe After Effects и Premiere Pro.

## Философия

Проект следует принципам **MVP (Minimum Viable Product)**:
- Минимум кода для достижения результата
- Последовательный pipeline без избыточных абстракций
- Автоматизация только критичных ручных операций
- Быстрая итерация и прямолинейное решение задач

## Структура проекта

```
osnovateli_doc_framework/
├── scripts/          # Скрипты обработки (выполняются последовательно)
├── assets/           # Шаблоны AE, эффекты, музыка, шрифты
└── data/            # Данные проектов (один проект = одно видео)
    └── {project_name}/
        ├── database/                  # Парсинг данных из Google Sheets
        ├── pictures/                  # Скачанные изображения
        ├── smart_cropped_pictures/    # Обрезанные изображения 16:9
        ├── photo_placeholder/         # MOV с эффектами для AE
        ├── video/                     # Скачанные видео
        ├── screenshots/               # Стоп-кадры из видео
        ├── voiceover/                 # Озвучка
        └── edit/                      # Финальные XML для монтажа
```

## Pipeline (последовательность выполнения)

### 1. Инициализация проекта
**Скрипт:** `0_structure.py`
- Создание структуры папок нового проекта
- Валидация формата названия `osnovateli_doc_{name}`

### 2. Парсинг исходных данных
**Скрипт:** `1_parse_links.py`
- Подключение к Google Sheets через Service Account
- Извлечение всех ссылок из колонки B
- Автоматическая категоризация: изображения, YouTube, видео, другое
- Сохранение в CSV файлы по категориям

**Скрипт:** `1.2_xml_placeholders.py`
- Парсинг дополнительных плейсхолдеров из XML

### 3. Обработка изображений
**Скрипт:** `2_download_img.py`
- Скачивание изображений из категоризированных ссылок
- Конвертация всех форматов в JPG
- Обработка Google Images короткие ссылок
- Создание заглушек для неудачных загрузок

**Скрипт:** `2.1_smart_cropping.py`
- Умная обрезка изображений в формат 16:9
- Детекция лиц и ключевых объектов
- Центрирование композиции

### 4. Обработка видео
**Скрипты:** `3_download_youtube_(na).py`, `3.1_download_other_video_(na).py`, `3.2_pulltube.py`, `4_pulltube_rename.py`
- Скачивание видео с YouTube и других платформ
- Переименование и нормализация файлов

### 5. Создание визуальных эффектов
**Скрипт:** `5_photo_placeholders.py`
- Конвертация JPG → MOV (ProRes 4444)
- Наложение эффекта царапин (`scratches_add.mp4`)
- Применение альфа-маски (`alpha_mask.mp4`)
- 10-секундные клипы с альфа-каналом для композитинга в AE

**Скрипт:** `5.1_Still_links_(na).py`
- Дополнительная обработка статичных изображений

### 6. Озвучка и финализация
**Скрипт:** `6_voiceover_(xml_error).py`
- Генерация или обработка озвучки
- Создание XML-таймлайна для импорта в Premiere Pro

## Технологический стек

**Python библиотеки:**
- `gspread` + `google-oauth2` — работа с Google Sheets API
- `requests` + `beautifulsoup4` — скачивание контента
- `selenium` + `webdriver-manager` — браузерная автоматизация
- `Pillow (PIL)` — обработка изображений
- `opencv-contrib-python` — умная обрезка и детекция
- `pydub` — работа с аудио

**Внешние инструменты:**
- `ffmpeg` — конвертация видео, создание эффектов
- Adobe After Effects — шаблоны композиций в `/assets/`
- Adobe Premiere Pro — финальный монтаж

## Интеграция с Google Sheets

Проект использует Service Account для доступа к таблицам:
1. Таблица должна быть расшарена на `CLIENT_EMAIL` из `.env`
2. Ссылки на медиа размещаются в колонке B
3. Скрипт сохраняет полную структуру таблицы в `input_gdoc.csv`

## Выходные данные

После прохождения всего pipeline получаем:

- Озвучку
- XML-таймлайн для импорта в Premiere Pro
- CSV-отчёты об ошибках и статистике

## Принципы работы

1. **Последовательность** — скрипты выполняются строго по порядку (0 → 1 → 2 → ... → 6)
2. **Идемпотентность** — повторный запуск скрипта безопасен (пропускает существующие файлы)
3. **Логирование** — ошибки записываются в CSV для ручной обработки
4. **Заглушки** — при ошибке загрузки создаётся placeholder с текстом ошибки
5. **Валидация** — проверка зависимостей, форматов, соотношений сторон
